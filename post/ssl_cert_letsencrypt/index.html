Securing web traffic with SSL/TLS certificates is essential for protecting data privacy and maintaining trust with users. In Kubernetes environments, managing SSL certificates manually can be cumbersome and error-prone. However, with tools like Cert Manager and Let‚Äôs Encrypt, obtaining and renewing SSL certificates can be automated, ensuring seamless encryption for your applications. In this blog post, we’ll explore how Cert Manager and Let‚Äôs Encrypt work together to provide automatic SSL certificate management in Kubernetes.

Cert Manager

Cert Manager is a Kubernetes native solution for managing SSL/TLS certificates. It automates the management of certificates, including issuance, renewal, and revocation. Cert Manager integrates with popular CAs and services like Let‚Äôs Encrypt, Venafi, and HashiCorp Vault. (Reference Issuer List: https://cert-manager.io/docs/configuration/issuers/)

letsencrypt

Let‚Äôs Encrypt is a free, automated, and open certificate authority. It provides SSL/TLS certificates for websites at no cost, making it ideal for Kubernetes environments. Let‚Äôs Encrypt certificates are valid for 90 days and can be renewed automatically.

Prerequisites

  1. Any cloud provider’s Kubernetes cluster -(GCP / EKS / AKS) or on-premise.
  2. A domain. ( You can get a free domain from freenom ).

Step 1 - Installing Cert-Manager

Apply the cert-manager manifest file for version 1.13 from the cert-manager GitHub releases page. Here’s an example command to apply the manifest file:

$ kubectl apply -f https://github.com/jetstack/cert-manager/releases/download/v1.13.0/cert-manager.yaml

Verify installation of cert-manager by checking the cert-manager namespace for running pods

$ kubectl get po -n cert-manager

cert-manager-cainjector-566zzxc-nmkwz     1/1     Running   0          3h

Step 2 - Create Issuer or Cluster Issuer

Apply the below manifest

apiVersion: cert-manager.io/v1
kind: ClusterIssuer 
metadata:
  name: letsencrypt-prod
  namespace: prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: zxy@zxy.com
    privateKeySecretRef:
      name: letsencrypt-prod
    solvers:
    - http01:
        ingress:
          class: nginx

Step 2 - Create Certificate

Apply the below manifest

apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: abc-net-cert
  namespace: prod
spec:
  secretName: abc-net-tls
  issuerRef:
    name: letsencrypt-prod
    kind: ClusterIssuer
  dnsNames:
  - abc.net

Step 3 - Create Certificate

Apply the below manifest

apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: abc-net-cert
  namespace: prod
spec:
  secretName: abc-net-tls
  issuerRef:
    name: letsencrypt-prod
    kind: ClusterIssuer
  dnsNames:
  - abc.net

Once we apply these YAML files, Cert-Manager will automatically request a certificate from Let’s Encrypt and store it in the specified Kubernetes Secret (abc-net-tls in this example).

Step 4 - Update Ingress

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: abc-net-ingress
  namespace: prod
spec:
  tls:
  - hosts:
    - abc.net
    secretName: abc-net-tls
  rules:
  - host: abc.net
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: <your-service>
            port:
              number: 80

Once we apply these changes, your Ingress controller will start serving your domain over HTTPS using the free SSL certificate from Let’s Encrypt.
